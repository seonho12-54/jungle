# sijun 작업 규칙

## 기본 원칙
- 테스트 코드는 반드시 작성한다.
- 테스트가 아닌 기능 코드는 `main.c`에만 작성한다.
- 코드는 최대한 단순하게 작성한다.
- 확장성보다 가볍고 이해하기 쉬운 구조를 우선한다.
- 빌드 산출물은 저장소 안에 만들지 않는다.
- 빌드와 테스트는 항상 `CMakePresets.json`의 `default` preset을 사용한다.
- 바이너리 출력 경로는 `/tmp/jungle-week6-sijun-build`를 유지한다.
- 직접 `sijun/build` 같은 저장소 내부 빌드 디렉터리를 새로 만들지 않는다.

## 기능 코드 규칙
- 구현하는 함수 위에는 주석을 작성한다.
- 함수 주석은 널리 쓰이는 `Doxygen` 형식을 사용한다.
- 함수 주석에는 역할, 인자, 리턴값을 짧게 설명한다.
- 예시는 다음과 같이 맞춘다.
```c
/**
 * @brief 함수의 역할을 짧게 설명한다.
 * @param arg 인자를 설명한다.
 * @return 리턴값을 설명한다.
 */
```

## 테스트 코드 규칙
- 테스트 함수 이름은 영어로 작성한다.
- 함수 이름만으로 부족한 시나리오는 주석으로 설명한다.
- 시나리오 주석은 다음 형태를 따른다.
  - `~일 때 ~하면 ~해야 한다.`
  - `~는 ~하다.`
- 테스트 본문은 주석으로 `given`, `when`, `then` 영역을 나눈다.
- 각 테스트는 한 가지 동작만 검증하도록 단순하게 유지한다.
